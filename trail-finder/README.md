WEB103 Project 1 - TrailFinder
Submitted by: Elvanice Previlma
About this web app: TrailFinder is a hiking trail guide that displays curated trails across the United States. Users can browse trails by difficulty, view key stats like length and elevation gain, and click into each trail for a full detail page with tips and scenery ratings.
Time spent: 2 days: 6 hours each day

<!--Required Features-->

The following required functionality is completed:

The web app uses only HTML, CSS, and JavaScript without a frontend framework
The web app displays a title
The web app displays at least five unique list items, each with at least three displayed attributes (such as title, text, and image)
The user can click on each item in the list to see a detailed view of it, including all database fields
A unique endpoint, such as localhost:3000/trails/angels-landing and localhost:3000/trails/half-dome

The web app serves an appropriate 404 page when no matching route is defined
The web app is styled using Picocss

The following optional features are implemented:

The web app displays items in a unique format, such as cards rather than lists or animated list items

Notes
Setting up the project environment was the biggest challenge. Getting Node.js and Express configured correctly on Windows took some troubleshooting. Once the server was running, understanding how Express routes work — especially how req.params.id connects the URL to the right trail data — was a helpful learning moment.
